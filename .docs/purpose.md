# Easy-Enhance Purpose

## 文件目的

本文件說明 Easy-Enhance 影像增強評估工作檯的建立目的、專案背景、系統定位、開發動機、使用情境與預期效益，作為整體專案的方向性文件。

## 專案背景

影像增強研究常需要反覆測試不同 enhancement method，然而每次接入新方法時，研究者往往都必須重新處理執行入口、資料夾結構、相依套件、輸出格式與評估流程。這使得大量時間被消耗在部署與串接，而非真正的模型分析與結果比較。

目前常見工具多半只處理單一面向：有些工具專注在 enhancement 模型訓練與推論，有些工具專注在 IQA 指標計算，但缺少一個以研究操作流程為中心、可統一整合 method、dataset、批次執行與結果視覺化的桌面工作檯。

## 開發動機

本專案的主要動機如下：

- 降低新方法接入時的重複部署成本。
- 統一資料集與方法的輸入規格。
- 提供批次測試、結果表格與指標圖表的一條龍工作流程。
- 建立可重複使用的研究基礎設施，而非一次性的實驗腳本。
- 讓研究者更容易比較不同 enhancement method 在同一資料集上的表現。

## 系統定位

Easy-Enhance 定位為研究流程整合平台。它不是新的影像增強模型，也不是單純的 IQA 指標函式庫，而是一個面向研究者日常測試流程的桌面評估工作檯。

它的角色介於以下兩者之間：

- Enhancement framework：提供方法實作與執行能力。
- IQA toolbox：提供品質評估與指標計算能力。

Easy-Enhance 的核心價值，在於將這兩者整合進一個統一的桌面 GUI 流程中，並進一步納入資料集匯入、任務排程、結果表格與圖表顯示。

## 系統目標

### 主要目標

- 建立以 CustomTkinter 為基礎的桌面 GUI。
- 支援 enhancement ZIP 與 dataset ZIP 的雙輸入流程。
- 建立標準化 plugin 介面以接入不同增強方法。
- 建立 queue-based 的批次處理與狀態追蹤流程。
- 自動計算 PSNR、SSIM、LPIPS、NIQE、EME、BRISQUE 六種指標。
- 輸出逐圖 metrics table 與六張指標趨勢圖。

### 次要目標

- 提升實驗流程的標準化程度。
- 提升研究結果的可追溯性與可比較性。
- 提供後續擴充更多 methods、metrics 與資料集格式的基礎。

## 使用對象

本系統主要使用對象包含：

- 影像處理研究者
- 電腦視覺專題學生
- 實驗室研究助理
- 需要大量比較 enhancement methods 的開發者

## 使用情境

典型使用流程如下：

1. 使用者匯入一個 enhancement method ZIP。
2. 使用者匯入一個 dataset ZIP。
3. 系統驗證 ZIP 結構並完成解壓。
4. 系統建立批次 jobs，逐張執行影像增強。
5. 系統計算六種品質指標並保存結果。
6. GUI 顯示 table、trend charts 與執行 log。
7. 使用者可匯出結果作為後續分析依據。

## 預期效益

Easy-Enhance 完成後，將能有效降低研究中的方法整合成本，讓使用者把更多時間投入在方法分析、實驗設計與結果比較，而非環境部署與流程重寫。

相較於單純腳本式做法，本系統提供：

- 更清楚的操作流程
- 更好的任務追蹤能力
- 更一致的輸入輸出規格
- 更直觀的結果視覺化方式
- 更容易被重複使用與擴充的系統骨架

## 結語

Easy-Enhance 的建立目的，是把影像增強研究從零散腳本與臨時串接，提升到具有標準化工作流與可維護結構的桌面研究工具層級。這份 purpose 文件定義了專案的必要性與價值，也是後續 spec 與 task 文件的基礎。
